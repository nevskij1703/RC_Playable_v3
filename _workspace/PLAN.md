# План работ по плейблу

## Цели из ТЗ

1. **Экономика продуктов** — добавить стоимость ингредиентов:
   | Ингредиент | Цена |
   |------------|------|
   | Лепёшка    | 3    |
   | Мясо       | 5    |
   | Помидоры   | 2    |
   | Картошка   | 3    |
   | Огурец     | 4    |
   | Сок        | 5    |

2. **UI / счётчики**
   - Счётчик монет (после каждого заказа добавляется стоимость блюда).
   - Счётчик обслуженных покупателей.
   - Адаптивная вёрстка (плейбл переставляет элементы при ресайзе окна — счётчики должны корректно вставать на всех разрешениях).

3. **Логика клиентов**
   - Всего на уровень: 20 покупателей.
   - Одновременно на сцене: максимум 3.
   - Каждому клиенту рандомно: либо 1, либо 3 заказа (блюда — рандом из всех вариантов).

4. **Переходы в Store** (2 способа)
   - Кнопка Install → переход.
   - Когда игрок отдаёт последнее блюдо последнему клиенту, блюдо НЕ выдаётся, а вместо этого — переход в Store.

5. **Smart Cooking System** (главный челлендж)
   - Нельзя приготовить блюдо, которое сейчас никому из текущих клиентов не нужно.
   - Игрок всегда может готовить только актуальные заказы.

6. **Hint / Tutorial System**
   - Если игрок выбирает неверный ингредиент / делает неверное действие — туториальный палец:
     - находит кратчайший путь к правильному действию;
     - показывает следующий шаг;
     - помогает быстрее собрать и отдать блюдо клиенту.

7. **Дополнительно** — фиксить любые баги, которые встретятся.

## Состояние проекта (на 2026-04-29)

- HTML собирается: `npm run build` → `./index.html` (~6.5 MB inline).
- Junction `node_modules/PlayableAdsBuilder` и `node_modules/PlayableAdsEngine` починены (Windows Junction → `./vendor/...`).
- Базовая логика клиентов и кухни уже есть в `src/js/displayObjects/location/`.

## Порядок работ (предложение)

1. Прочитать ключевые модули: `Location.js`, `PlayableController.js`, `Character.js`, `Food.js`, `Dish.js`.
2. Реализовать **счётчики и экономику** (наименее рискованно, видимый прогресс).
3. Привести **логику клиентов** к ТЗ (20/3, 1 или 3 заказа).
4. Сделать **Smart Cooking** — главный челлендж: блокировать сборку нерелевантных блюд.
5. Доработать **Hint/Tutorial** — указатель на ближайший шаг.
6. Сделать **финальный заказ → Store**.
7. Прогон, фикс багов.
